# Screenshots

Drop PNGs here named exactly as below; they're referenced from the project README.

| Filename | What to capture | Suggested width |
|---|---|---|
| `river.png` | River view with at least 2–3 expanded memory cards visible, showing tag chips, change badge, flow strip | 1400 |
| `git-tree.png` | The sticky horizontal git tree with keyword labels showing | 1400 |
| `graph.png` | Constellation graph view with clusters and flow arrows on | 1400 |
| `summary.png` | Project summary dashboard — hero card + stats grid + donut + heatmap visible | 1400 |
| `diff.png` | An expanded memory card with the GitHub-style diff open and syntax-highlighted | 1100 |

How to capture cleanly:
1. `chronicle serve` on a real project of yours (or this one — Chronicle observes itself).
2. Open in Chrome at 1400×900. Cmd+Shift+P → "Capture full size screenshot" (devtools) or use a tool like CleanShot.
3. Drop the PNGs in this directory; the README image refs will pick them up.

PNGs in this dir are referenced via `assets/screenshots/<file>.png` from the project README.
